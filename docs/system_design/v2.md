# System Design — Basic v2

---

## What Changes from v1

v2 does not replace v1. It extends the existing platform by adding **per-negotiation mode selection** to both the dealer and buyer agents. Every other part of the system — KA matching, shortlist, AID exchange, sequential fallback, fee recording — is unchanged.

**The single new capability:** before each negotiation begins, each agent independently selects either **Manual** or **Auto** mode. Any combination across DA and BA is valid.

| DA Mode | BA Mode | Label | Description |
|---|---|---|---|
| Manual | Manual | Manual–Manual | Both humans negotiate. Functionally identical to v1 negotiation behaviour. |
| Manual | Auto | Manual–Auto | Human dealer reads and responds to offers from an automated buyer. |
| Auto | Manual | Auto–Manual | Automated dealer negotiates against a human buyer. |
| Auto | Auto | Full Auto | Both agents automated. Full simulation mode for controlled experiments and report analysis. |

In **Manual** mode the agent behaves exactly as in v1: the GUI waits for human input, the human reads each offer and types a counter. In **Auto** mode the agent runs the Faratin concession formula with the regression prediction layer — no human input during the negotiation.

---

## Mode Selection — When and How

Mode is selected **per negotiation**, in the Negotiation tab, before the first offer is sent. Once the first offer is sent or received, the mode is locked for that negotiation. Agents can switch modes between separate negotiations within the same session.

**Why per-negotiation (not per-session):**
- Maximises flexibility for the report's experimentation and demo scenarios — the same session can run a Manual–Auto negotiation followed by a Full Auto negotiation for direct comparison
- Prevents agents from being locked into a mode that makes a particular demo awkward
- Per-session mode would require restarting agents to change strategy, which wastes demo time

**DA mode selection flow:**
```
DA receives buyer interest (KA INFORM) → Negotiation tab becomes visible
  → DA reviews buyer AID and first offer
  → DA clicks Accept or Decline

If Accept:
  → "Negotiation Setup" panel appears in Negotiation tab
  → DA selects mode: [Manual] [Auto]
  → If Auto: α slider appears (default 0.7), locked on confirm
  → DA clicks Confirm → first CFP sent at retail price → negotiation begins
```

**BA mode selection flow:**
```
BA receives DA's AID from KA (AID exchange INFORM) → Negotiation tab becomes visible
  → "Negotiation Setup" panel appears
  → BA selects mode: [Manual] [Auto]
  → If Auto: α slider appears (default 1.0), locked on confirm
  → BA clicks Confirm → BA ready, waits for DA's opening CFP
```

**Why α is entered in the Negotiation tab (not the Listings tab for DA):**
α is a negotiation strategy parameter, not a car attribute. The same car could be negotiated with a different α against different buyers in different sessions. Entering α at listing time would lock a single value per car, making per-negotiation mode selection meaningless. The Negotiation tab is the correct and consistent location for both agents.

## v2 System Flow (End to End)

```
Steps 1–7 are identical to v1.
(DA registers listings → BA searches → KA matches → BA shortlists → KA contacts DA → DA accepts → AID exchange)

8. Mode selection (NEW in v2)
        → Both DA and BA independently select Manual or Auto in their Negotiation tab
        → If Auto: α is entered and confirmed
        → Mode is locked once the first offer is sent

9. DA sends opening CFP at retail price (round 0)

10. Each round — diverges by mode:

    If Auto (either agent):
      → receives opponent offer
      → RegressionPredictor.addPoint(round, opponentOffer)
      → TimeBasedStrategy.calculateNextOffer(state) → myOffer
      → enhanced accept condition checked (Faratin base + regression layer)
      → if accept: send ACCEPT_PROPOSAL, notify KA
      → if not accept: send PROPOSE with calculated offer

    If Manual (either agent):
      → GUI receives opponent offer, displays it in offer history table
      → round counter updates
      → human reads offer, types counter-offer, clicks Counter / Accept / Walk Away
      → response sent based on human input
      → BA: system enforces reserve price privately (rejects input above reserve)
      → DA: system enforces floor price privately (rejects input below floor)

11. Negotiation ends (any mode combination):
        → deal reached: ACCEPT_PROPOSAL sent by either agent
        → round limit reached: REJECT_PROPOSAL sent, deal-failed INFORM to KA
        → walk-away: either agent sends REJECT_PROPOSAL (isFinal = true)
        → no-overlap detected (Auto mode only): regression triggers early walk-away

12. Deal outcome handling identical to v1.
```

## Auto Mode: Concession Strategy — Faratin Time-Based Formula

*This section applies only when an agent is in Auto mode.*

Every automated offer is produced by the Faratin et al. (1998) time-based concession formula:

```
factor(t, T, α) = ( t / T ) ^ ( 1 / α )

DA offer at round t:
  = floorPrice + (retailPrice − floorPrice) × (1 − factor)
  Starts at retailPrice, moves down toward floorPrice as rounds increase.

BA offer at round t:
  = firstOffer + (reservePrice − firstOffer) × factor
  Starts at firstOffer, moves up toward reservePrice as rounds increase.
```

**α — the personality parameter:**

| α value | Behaviour | Profile |
|---|---|---|
| < 1 | Large concessions early, slows near limit | Conceder |
| = 1 | Constant movement every round | Linear |
| > 1 | Holds near opening offer, big jump at end | Boulware |

**Default α values:**
- DA default: `0.7` (conceder-leaning — per Dr. Lee's directive to push toward faster closure)
- BA default: `1.0` (linear — neutral, balanced concession)
- Both entered in the Negotiation tab before negotiation begins; locked once first offer is sent

All offers are bounded: Auto DA never goes below `floorPrice`; Auto BA never exceeds `reservePrice`. Private limits are enforced locally and never transmitted.

In **Manual mode**, the human sets their own offer each round. The system still enforces the private limit as a hard floor/ceiling on manual input (rejects values beyond the limit), but the human controls all offer amounts within that range.

## Auto Mode: Prediction Algorithm — Linear Regression

*This section applies only when an agent is in Auto mode.*

As offers are exchanged, the auto agent fits a linear regression on the opponent's offer history to predict their approximate limit at the deadline.

**How it works:**
```
x = round number,  y = opponent offer at that round
Fit: y = a + b × x  (least-squares, no library needed)
Predict: estimated opponent limit = a + b × T
```

**Reliability gates — all three must pass before prediction is used:**

| Gate | Config key | Default | Purpose |
|---|---|---|---|
| Min data points | `negotiation.regressionMinPoints` | 3 | Too few points = meaningless fit |
| R² threshold | `negotiation.regressionMinR2` | 0.80 | Low R² = noisy/non-linear data; fall back to Faratin |
| Slope threshold | `negotiation.boulwareSlopeThreshold` | 50.0 RM/round | Near-zero slope = Boulware detected; regression suppressed |

**Bounds clamping:** predicted value is clamped to the opponent's observed offer range. Cannot extrapolate to a value above or below what the opponent has already offered.

**How the prediction influences decisions:**

| Agent in Auto | If prediction is reliable | Action |
|---|---|---|
| BA watching DA | Predicted DA limit ≤ my current Faratin offer | Accept now — they're nearly at my price |
| DA watching BA | Predicted BA limit ≥ my current Faratin offer | Hold or make final aggressive move |
| Either | Predicted DA limit > predicted BA limit | No-overlap — walk away early |

The regression layer does **not** replace Faratin. It enhances the accept condition and enables early termination.

## Auto Mode: DA Shortening Strategy

*This section applies only when DA is in Auto mode.*

Per Dr. Lee's directive, the auto dealer is designed to push toward faster deal closure through three mechanisms:

**Mechanism 1 — Conceder-leaning α (default 0.7):**
Large early concessions signal deal attractiveness to the opposing agent (or human). Creates an incentive to accept sooner rather than waiting out the round limit.

**Mechanism 2 — Regression-driven final commitment:**
When the regression predictor estimates BA's predicted ceiling is within `negotiation.bestOfferThreshold` (default RM500) of DA's current Faratin offer, DA makes one aggressive final move to just above the predicted ceiling rather than continuing incremental concessions.

**Mechanism 3 — Early walk-away on predicted no-overlap:**
If regression predicts BA will never reach DA's floor price, DA exits immediately rather than grinding to the round limit.

---

## Boulware Handling (Auto Mode)

When an opponent — whether Auto or Manual — produces nearly flat offers (slope near zero), the regression is suppressed (slope gate fails). The auto agent falls back to pure Faratin: continues conceding on its own schedule, holds position, and waits for the opponent's eventual movement. The opponent's late move will cross the agent's current offer and trigger the base accept condition naturally.

This also means: a **Manual human playing Boulware** (typing barely-changing offers) is correctly handled by the same mechanism — the regression slope will be near zero, suppression fires, and the auto agent proceeds normally.

## v2 GUI Changes from v1

**Both DA and BA Negotiation tabs — new "Negotiation Setup" panel:**

Appears at the top of the Negotiation tab once a negotiation is imminent (after DA accepts buyer interest / after BA receives AID exchange). Disappears once the first offer is sent.

Contents:
- Mode toggle: `[ Manual ]  [ Auto ]`
- If Auto selected:
  - α slider (range 0.1 – 10.0, default per agent type)
  - α value label showing current selection and profile name (Conceder / Linear / Boulware)
- Confirm button to lock mode and begin

**BA Window — Negotiation tab, Auto mode:**
- Auto-mode banner: "AUTO MODE — negotiating automatically"
- Live display: current round, last sent offer, last received offer, regression prediction (if active and reliable)
- Offer history table auto-populates each round
- No manual counter-offer input field

**BA Window — Negotiation tab, Manual mode:**
- Identical to v1 BA negotiation tab
- Counter-offer input field + Accept, Counter, Walk Away buttons
- Reserve price enforced as hard ceiling on input

**DA Window — Negotiation tab, Auto mode:**
- Auto-mode banner: "AUTO MODE — negotiating automatically"
- Same live display as BA auto mode
- Floor price visible to DA in status area (never shown to BA)

**DA Window — Negotiation tab, Manual mode:**
- Identical to v1 DA negotiation tab
- Counter-offer input field + Accept, Walk Away buttons
- Floor price enforced as hard floor on input

## v2 Cases and Edge Cases

### Mode Combination Outcomes

| DA Mode | BA Mode | Expected behaviour |
|---|---|---|
| Auto | Auto | Full simulation — Faratin curves play out; regression may trigger early accept or early walk-away |
| Auto | Manual | Auto DA concedes on schedule; human BA reads each offer and responds manually |
| Manual | Auto | Human DA reads each auto BA offer and responds manually; regression predicts human DA's trend if offers are consistent |
| Manual | Manual | Identical to v1 — no automation fires |

### Normal Outcomes

| Case | Situation | Outcome |
|---|---|---|
| Deal reached (auto) | Faratin or regression accept condition fires | ACCEPT_PROPOSAL sent; deal recorded; KA collects commission |
| Deal reached (manual) | Human clicks Accept | ACCEPT_PROPOSAL sent; deal recorded |
| Round limit reached | No accept condition fires within T rounds | REJECT_PROPOSAL; deal-failed INFORM to KA; next dealer in shortlist |
| No-overlap early exit (auto) | Regression predicts deal impossible | Walk-away sent; deal-failed INFORM to KA; next dealer in shortlist |

### v2-Specific Edge Cases

| Case | Mode | Situation | Handling |
|---|---|---|---|
| Boulware opponent | Any auto agent vs opponent | Opponent slope near zero | Regression suppressed; Faratin base continues |
| Low R² (noisy offers) | Auto | Opponent offers non-monotonic | R² gate fails; Faratin base only |
| Insufficient history | Auto | < 3 rounds elapsed | Regression does not fire |
| Human enters offer above reserve | Manual BA | GUI input exceeds reserve | System rejects input, shows warning |
| Human enters offer below floor | Manual DA | GUI input below floor | System rejects input, shows warning |

## Simulation Parameters (Auto Mode)

When both agents are in Auto mode, the negotiation is fully deterministic given these inputs. These are the variables for the report's critical analysis experiments:

| Parameter | Config key | Default | Controls |
|---|---|---|---|
| DA concession curve | `negotiation.alphaDealer` | 0.7 | Conceder-leaning by default |
| BA concession curve | `negotiation.alphaBuyer` | 1.0 | Linear by default |
| Round limit | `negotiation.maxRounds` | 20 | Deadline T |
| Regression min data | `negotiation.regressionMinPoints` | 3 | When regression fires |
| R² confidence | `negotiation.regressionMinR2` | 0.80 | When regression is trusted |
| Boulware detection | `negotiation.boulwareSlopeThreshold` | 50.0 | Slope below which regression suppresses |
| DA final push gap | `negotiation.bestOfferThreshold` | 500.0 | RM gap triggering DA's final commitment |

**Suggested experiments for the report:**

| Experiment | DA | BA | Expected result |
|---|---|---|---|
| Full auto, both Conceder | Auto α=0.5 | Auto α=0.5 | Fast deal, early rounds |
| Full auto, both Boulware | Auto α=8.0 | Auto α=8.0 | No deal, or barely closes at round T |
| Full auto, DA Conceder vs BA Boulware | Auto α=0.7 | Auto α=8.0 | Deal near DA floor; BA extracts most value |
| Auto DA vs Manual BA (human) | Auto α=0.7 | Manual | Human observes auto DA conceding; report on human decision-making vs auto outcome |
| Regression ON vs OFF | Auto α=0.7 | Auto α=0.7 | Compare rounds taken and deal price |
| No-overlap scenario | Auto | Auto | Regression-driven early exit vs grinding to T |
