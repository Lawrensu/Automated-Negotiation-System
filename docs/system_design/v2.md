# System Design — Basic v2

---

## What Changes from v1

In v1, both the buyer and dealer are human-operated. The system facilitates communication between two humans via GUI.

In v2, **the buyer agent becomes fully automated**. The BA no longer waits for human input during negotiation, instead, it calculates and sends offers autonomously using a concession strategy and a prediction layer. The dealer agent is also automated with a configurable strategy, enabling controlled simulation and the critical analysis required by the project report.

The rest of the system (KA matching, shortlist, AID exchange, sequential fallback) is **unchanged from v1**.

---

## v2 System Flow (End to End)

```
Steps 1–7 are identical to v1.
(DA registers listings → BA searches → KA matches → BA sends shortlist → KA contacts DA → DA accepts → AID exchange)

8. DA contacts BA directly
        → DA sends opening CFP at retail price (round 0)

9. Each round (automated):
        BA side:
          → receives DA offer
          → RegressionPredictor updates with (round, DA offer)
          → TimeBasedStrategy calculates BA's next offer (Faratin formula)
          → enhanced accept condition checked (see below)
          → if accept: send ACCEPT_PROPOSAL, notify KA
          → if not accept: send PROPOSE with calculated offer

        DA side:
          → receives BA offer
          → RegressionPredictor updates with (round, BA offer)
          → TimeBasedStrategy calculates DA's next offer (Faratin formula, conceder-leaning α)
          → enhanced accept condition checked
          → if accept: send ACCEPT_PROPOSAL, notify KA
          → if not accept: send PROPOSE with calculated offer

10. Negotiation ends:
        → deal reached: either agent's accept condition fires
        → no-overlap detected: regression predicts deal is impossible → early walk-away
        → round limit reached: negotiation ends with no deal

11–12. Deal outcome handling identical to v1.
```

## Concession Strategy : Faratin Time-Based Formula

Every automated offer is produced by the Faratin et al. (1998) time-based concession formula:

```
factor(t, T, α) = ( t / T ) ^ ( 1 / α )

DA offer at round t:
  = floorPrice + (retailPrice − floorPrice) × (1 − factor)
  Starts at retailPrice, moves down toward floorPrice as rounds increase.

BA offer at round t:
  = firstOffer + (reservePrice − firstOffer) × factor
  Starts at firstOffer, moves up toward reservePrice as rounds increase.
```

**α (alpha)  the personality parameter:**

| α value | Behaviour | Profile |
|---|---|---|
| < 1 | Large concessions early, slows near limit | Conceder |
| = 1 | Constant movement every round | Linear |
| > 1 | Holds near opening offer, big jump at end | Boulware |

**Default α values:**
- DA default: `0.7` (conceder-leaning — per Dr. Lee's directive to push toward faster closure)
- BA default: `1.0` (linear — neutral, balanced concession)
- Both are configurable via GUI before each negotiation

All offers are bounded: DA never goes below `floorPrice`; BA never exceeds `reservePrice`. These limits are enforced privately and never transmitted.

## Prediction Algorithm : Linear Regression

As offers are exchanged, each party's offer sequence reveals information about where their offers are heading. A linear regression on the opponent's offer history predicts their approximate limit at the deadline.

**How it works:**
```
x-axis: round number
y-axis: opponent offer amount
Fit: y = a + b × x  (least-squares)
Predict: estimated opponent limit = a + b × T
```

**When it fires:** Only when the following three gates all pass:
1. `history.size() >= negotiation.regressionMinPoints` (default 3) : not enough data below this
2. `|slope| >= negotiation.boulwareSlopeThreshold` (default 50 RM/round) : slope near zero means opponent is Boulware and regression is not informative; fall back to Faratin only
3. `R² >= negotiation.regressionMinR2` (default 0.80) : low R² means noisy/non-linear data; fall back to Faratin only

**Bounds clamping:** The predicted value is clamped to the physically possible range. For a DA prediction, the result cannot exceed the opponent's first observed offer. For a BA prediction, the result cannot fall below the opponent's first observed offer.

**How the prediction is used:**

| Agent | Prediction use |
|---|---|
| BA watching DA offers | If predicted DA limit ≤ BA's current Faratin offer → accept now (they're nearly at my price) |
| DA watching BA offers | If predicted BA limit ≥ DA's current Faratin offer → hold or make one final aggressive offer |
| Either agent | If predicted DA limit > predicted BA limit → no-overlap zone → walk away early |

The prediction layer does **not** replace the Faratin formula. It adds an enhanced accept condition on top of the base strategy.

## DA Shortening Strategy

Per Dr. Lee's directive, the dealer agent is designed to push toward faster deal closure rather than grinding to the round limit. This is achieved through two mechanisms:

**Mechanism 1 : Conceder-leaning α (default 0.7):**
The DA makes larger concessions in early rounds, signalling deal attractiveness to BA. BA faces a "lock it in now" incentive as the DA's offers are visibly moving.

**Mechanism 2 : Regression-driven early commitment:**
When the regression predictor estimates that BA's predicted ceiling is close to DA's current Faratin offer (within `negotiation.bestOfferThreshold`, default RM500), DA makes one final aggressive move to just above the predicted BA ceiling. This collapses the remaining gap rather than conceding incrementally.

**Mechanism 3 : Early walk-away on no-overlap:**
If DA's regression (watching BA's offers) predicts BA will never reach DA's floor price, DA walks away early. This saves the remaining rounds without any possibility of a deal, rather than grinding to the limit.

---

## Boulware Handling

When an opponent uses a Boulware strategy (α >> 1), their offers barely move across rounds. The regression slope is near-zero. If this were used naively, the predictor would mistake "not moving yet" for "nearly at limit" and cause the agent to over-concede.

**Detection and response:**
- If `|slope| < boulwareSlopeThreshold` → regression is suppressed for this round
- Agent falls back to pure Faratin base strategy: continue calculating offers normally, hold own position, wait for the opponent's late-round movement
- Boulware opponent's late-round jump will eventually cross the agent's current offer → accept condition fires naturally via Faratin base

This means a Boulware buyer against a Conceder dealer is valid game-theoretic behaviour (patient party extracts more value) — the system handles it correctly, does not misfire, and the outcome is appropriate for critical analysis in the report.

## v2 Cases and Edge Cases

### Normal Outcomes (automated)

| Case | Situation | Outcome |
|---|---|---|
| Deal reached via Faratin base | Opponent offer crosses agent's own calculated offer | Accept fires; deal recorded |
| Deal reached via regression | Regression predicts opponent limit ≤ agent's current offer | Accept fires early, saving remaining rounds |
| Round limit reached | Neither accept condition fires within T rounds | No deal; KA notified; KA tries next dealer in shortlist |

### v2-Specific Edge Cases

| Case | Situation | Handling |
|---|---|---|
| Boulware opponent | Opponent slope near zero; regression suppressed | Fall back to Faratin base; wait for late-round movement |
| No-overlap detected | Predicted DA limit > predicted BA limit; deal mathematically impossible | Agent walks away early; KA moves to next dealer |
| Insufficient data | history.size() < regressionMinPoints | Regression does not fire; Faratin base only |
| Low R² (noisy data) | Opponent offers are non-monotonic or oscillating | R² gate fails; Faratin base only |
| BA's firstOffer below DA floor | No negotiation zone exists from the start | DA rejects immediately (same as v1) |

## Simulation Parameters

Since both agents are automated in v2, the negotiation is fully deterministic given these inputs. These are the variables for the report's critical analysis experiments:

| Parameter | Controls | Default |
|---|---|---|
| `negotiation.alphaDealer` | DA concession curve shape | 0.7 (Conceder) |
| `negotiation.alphaBuyer` | BA concession curve shape | 1.0 (Linear) |
| `negotiation.maxRounds` | Deadline T | 20 |
| `negotiation.regressionMinPoints` | Minimum offers before regression fires | 3 |
| `negotiation.regressionMinR2` | Confidence gate for regression | 0.80 |
| `negotiation.boulwareSlopeThreshold` | Slope below which regression is suppressed | 50.0 (RM/round) |
| `negotiation.bestOfferThreshold` | Gap at which DA makes final aggressive move | 500.0 (RM) |

**Suggested experiments for the report:**

| Experiment | DA α | BA α | Expected result |
|---|---|---|---|
| Both Conceder | 0.5 | 0.5 | Fast deal, early rounds |
| Both Boulware | 8.0 | 8.0 | No deal, or barely closes at round T |
| Conceder DA vs Boulware BA | 0.7 | 8.0 | Deal near DA floor; BA extracts most value |
| Boulware DA vs Conceder BA | 8.0 | 0.5 | Deal near BA reserve; DA holds value |
| Regression ON vs OFF | 0.7 | 0.7 | Show rounds saved and outcome quality difference |
| No-overlap scenario | any | any | Regression-driven early exit vs grinding to round T |
